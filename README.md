
---
Minecraft 数据包 指令商店 V1.3 | 基于Homestead1.20.1开发，非Homestead请确保陶罐item字段可用

这个数据包允许你使用'指令宏剑'(注意，请尽可能把指令宏剑的颜色往Minecraft的aqua这个颜色上靠)创建并使用‘指令商店’

---
'指令商店'是一个被代理的命令方块，被代理的命令将变成商品，需支付绿宝石才能使用。

'指令商店'支持两种方式交易方式：
一、直接交易，方式是使用绿宝石直接左键'指令商店'
二、间接交易，方式是使用红石信号触发，'指令商店'被激发后会自动搜索最近的'命令执行器'，在'命令执行器'上方陶罐余额足够的情况下，执行指令。这种方式执行的指令，如果执行失败，不会扣除余额。

注意：
一、'指令商店'任何情况下都能交易，但如果缺乏'命令执行器'，则执行一定失败。这意味着，在缺乏'命令执行器'的情况下使用方式一进行交易，等于把绿宝石扔进垃圾桶。
二、'指令商店'的执行者和执行位置，默认为'命令执行器'，这是可以调整的，详见下文的'接口&细节'。

使用绿宝石右键'指令商店'，可调整'指令商店'的价格。

使用方式一与'指令商店'进行交易，可锁定价格，若要重新调整价格，必须重置'指令商店'。

---
如何创建'指令商店'？
使用'指令宏剑'左键点击命令方块

如何创建'命令执行器'？
使用'指令宏剑'左键点击空陶罐，或者装着绿宝石的陶罐

---
接口&细节：
一、代理命令在执行的生命周期内，会在'指令商店'原地生成一个minecraft:marker，你可以使用@e[tag=.]索引到它，生命周期结束后，它会消失。
二、代理命令在执行的生命周期内，会给使用方式一与'指令商店'直接交易的玩家，添加一个'..'tag，你可以使用@p[tag=..]索引到交易玩家，生命周期结束后，该tag会消失。
三、被充能的'指令商店'无法被其他红石脉冲触发，但任然可以通过绿宝石直接手动交易。
四、被充能的'命令执行器'将无法执行任何指令。
五、'指令宏剑'认主，丢弃或死亡导致'指令宏剑'变成掉落物，会导致它自动传送回主人的背包。不过，你可以主动将它放进容器或其他可置物的方块中，这将断开你与它之间的链接。如果你不慎丢失'指令宏剑'，你可以/function command_packer_core:set_up或者/reload重新获得一把。
六、主手持有'指令宏剑'，同时副手持有绿宝石，会临时将你的模式更改为创造模式。
七、创造模式下会持续检查你是否至少有一只手持有'指令宏剑'，如否，你的模式将自动更改回生存模式。
